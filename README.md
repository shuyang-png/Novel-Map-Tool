# Novel-Map-Tool

纯前端小说地图编辑工具，HTML + JS 模块，打开即用。JSON 文件为核心数据，HTML 为渲染器/编辑器。

## ⚠️ 浏览器要求

> **必须使用 Chrome 86+ 或 Edge 86+**，不支持 Firefox、Safari、移动端浏览器。
>
> 本项目使用 JS 模块化加载（多个 `<script>` 标签），依赖现代浏览器对 ES6 模块、Canvas 2D、File System Access API 的完整支持。Firefox 和 Safari 的 File System Access API 支持不完整，移动端浏览器无法处理鼠标事件。

## 功能列表

### 地图画布
- 2000×2000 像素画布，带网格线和坐标轴
- 鼠标滚轮缩放（0.1x ~ 2x）
- 鼠标拖拽平移
- 自适应窗口大小

### 坐标点
- 点击地图直接添加坐标点（名称 + 备注）
- 左侧面板手动输入坐标添加（X/Y 0~2000）
- **同一坐标 (x, y) 不能重复添加**，重复时会弹窗提示并阻止
- 坐标点列表：查看、删除
- 点击"查看"自动平移定位：地图平滑滑动（300ms ease-out 动画）将坐标点移到画布居中，再弹出备注框
- 点击坐标点弹出详情框，支持在线编辑备注（≤200字），取消编辑自动恢复原始内容

### 地理标识（V2 — 26 种预设）

地理标识用于在地图上标记各类地理特征，每种预设绑定视觉符号，不只靠颜色区分。

**26 种预设，7 个分组：**

| 分组 | 预设 |
|------|------|
| 通用 | 通用（无约束，自由选择形状） |
| 水域 | 河流、湖泊、海洋、泉眼 |
| 植被 | 森林、沼泽、草原、沙漠、竹林 |
| 地形 | 山脉、悬崖、火山、雪峰、洞穴 |
| 建筑/人文 | 城市、城堡、村庄、庙宇、废墟 |
| 道路/边界 | 道路、城墙、行军路线、桥梁 |
| 范围/区域 | 王国、营地 |

**5 种底层形状：**
- 折线（polyline）— 线状特征
- 多边形（polygon）— 不规则区域
- 矩形（rect）— 规则区域
- 椭圆（ellipse）— 规则区域
- 标注点（pin）— 具体位置

**形状约束：**
- 线状特征（河流、山脉、道路等）只能使用折线
- 面状特征不能使用折线，但椭圆/矩形/多边形自由选择
- 标注点对所有类型开放

**Pin 标注点放置：**
- 选择标注点后，直接输入 X/Y 坐标，点击「添加地理标识」即可放置
- 与其他地理标识（矩形、椭圆、折线等）操作方式一致
- 左侧列表中显示标注点的 (x, y) 坐标

**面积与长度自动计算：**
- 添加地理标识时自动计算：折线 → 长度，矩形/椭圆/多边形 → 面积
- 地图渲染和左侧列表中均显示计算结果
- 基于 `unit` 单位自动格式化（如"约12万平方里"）

**预设自动填充：**
- 选择预设类型后，描边色、线宽、粗线、内部符号自动填入
- 用户可手动覆盖任何样式参数

**视觉符号（Canvas 几何绘制，无字体依赖）：**
- 河流 → 沿线方向箭头
- 森林 → 散布小树（△ + 竖线）
- 湖泊 → 水平波浪线
- 城墙 → 实线 + T 形止标
- 营地 → 帐篷形（△ + 旗子）
- 城市 → 十字街道网格
- 泉眼（pin）→ 水滴形
- 火山（pin）→ 火山形
- 雪峰（pin）→ 雪山形
- 洞穴（pin）→ 拱门形
- 村庄（pin）→ 人字屋顶小屋
- 庙宇（pin）→ 宝塔形
- 废墟（pin）→ 断柱形
- 等等……每种预设都有独特的内部符号

**交互体验：**
- **鼠标悬停弹窗**：鼠标移到任何地理标识上，弹出名称浮窗 + 类型 + 坐标/尺寸/面积信息
- **地理标识定位**：每个标识的"定位"按钮可将地图平移到其中心点（矩形中心、椭圆圆心、pin 坐标、多边形重心）

### 地图关联
- 设置当前地图与目标地图的双向关联
- 关联点在地图上以 🚪 传送门标记显示，附目标地图名称
- 自动在目标地图中创建反向关联（双写到两个 JSON 文件）
- 支持删除关联（同步删除反向并更新两个 JSON）

### 量化单位
- 自定义单位名称（如"里"）
- 自定义单位说明（如"1里=500米"）
- 显示在地图左下角

### 文件操作
- **📄 新建地图**：不刷新页面即可创建空白地图，输入名称后画布自动重置，可直接开始编辑
- **💾 保存地图**：保存当前地图（有工作目录时直写文件，无目录时降级下载）
- **读取地图**：从已保存地图列表或工作目录加载地图数据
- **清空所有数据**：重置画布

### 版本管理
- **创建版本**：为当前地图状态创建快照（支持填写章节和标签）
- **版本时间轴**：左侧面板显示所有版本，点击可回溯查看任意历史版本
- **版本查看模式**：查看历史版本时禁止编辑，可随时退出回到最新版本

### 工作目录（File System Access API）
- 选择本地文件夹作为工作目录（需 Chrome/Edge 86+）
- 保存直接写入目录文件，不再触发浏览器下载
- 目录中的 JSON 文件自动列出，点击即可加载
- 目录选择状态持久化，刷新页面后自动恢复
- 不支持的浏览器自动降级到传统下载/上传方式

### 已保存地图管理
- 左侧面板显示已保存地图列表
- 一键加载/删除已保存的地图

## 使用方式

直接将 `小说地图工具.html` 拖入 Chrome/Edge 浏览器打开，或通过文件菜单打开。**不支持 Firefox、Safari 及其他浏览器。**

> **为什么必须是 Chrome/Edge？**
> - 本项目采用 JS 模块化架构（多个 `<script>` 标签按顺序加载），依赖完整的 ES6 支持
> - File System Access API（工作目录直读直写）目前仅 Chrome 86+ 和 Edge 86+ 完整支持
> - Canvas 2D 的复杂交互（拖拽、缩放、悬停检测）在 Firefox/Safari 上行为不一致
> - IndexedDB 用于持久化目录句柄，部分浏览器实现有差异
>
> **不要在 Firefox、Safari、移动端浏览器上尝试使用此工具。**

**完全离线可用**：HTML + JS 文件在本地即可运行，零外部依赖，不发起任何网络请求。地图数据以 JSON 文件保存在本地。

## 在 Coze 上使用（AI 智能建图）

本项目包含一个 **novel-map-api** Skill，用于 Agent（如 Coze）通过编程 API 生成地图 JSON，无需手写。

### Skill 安装到 Coze

1. **下载 skill 包**：从 GitHub 仓库的 `skill/` 目录获取 `novel-map-api.skill` 文件
2. **在 Coze 平台上传**：Bot 配置 → 技能 → 安装技能 → 上传 `.skill` 文件
3. **配置工作目录**：Agent 生成的地图 JSON 将保存在 `/root/.openclaw/workspace/Novel-Map-Tool/` 目录下
4. **加载 HTML 渲查看**：用浏览器打开 `小说地图工具.html`，加载生成的 `.json` 文件即可查看/编辑

### Coze 使用流程

```
用户: "创建一份玄幻小说地图"
→ Agent 调用 novel-map-api 生成 世界地图
→ 用户: "第2章，主角故乡被毁"
→ Agent loadMap → searchGeo → editGeo → createVersion → saveMap
```

### Skill 约束说明

novel-map-api 的 SKILL.md 包含以下关键约束，Agent 会自动遵守：
- 坐标点不可重复（同一 (x,y) 只能有一个）
- 实体建筑不重叠（除非作者要求）
- 湖泊/海洋/王国默认圆形（ellipse）
- 版本更新时必须先修改已有元素再调用 createVersion

### 文件加载模式

| 场景 | Chrome/Edge | 其他浏览器 |
|------|-------------|-----------|
| 工作目录 | 直读直写（选择文件夹后自动保存） | ❌ 不支持 |
| 无目录 | Download（下载到默认位置） | Download |
| 加载已有地图 | 从目录列表中选择 或 上传文件 | 仅上传 |

## 数据格式

地图数据以 JSON 保存，核心结构：

```json
{
  "mapName": "主世界地图",
  "scale": 1,
  "mapSize": 2000,
  "unit": { "name": "里", "desc": "1里=500米" },
  "notes": [{ "id": "note_xxx", "name": "长安城", "x": 1000, "y": 800, "content": "帝都" }],
  "geoMarkers": [
    { "id": "geo_xxx", "presetType": "河流", "type": "polyline", "name": "青龙河",
      "points": [{"x":200,"y":100},{"x":500,"y":300}],
      "style": { "strokeColor": "#5dade2", "lineWidth": 3, "bold": false, "symbol": "arrows" } },
    { "id": "geo_yyy", "presetType": "城市", "type": "rect", "name": "长安城",
      "x": 800, "y": 600, "width": 200, "height": 150,
      "style": { "strokeColor": "#e74c3c", "lineWidth": 2, "fillColor": "rgba(231,76,60,0.12)", "bold": true, "symbol": "grid" } }
  ],
  "mapRelations": [{ "id": "relation_xxx", "currentMapName": "主世界", "currentXY": "200,1800", "targetMapName": "火焰秘境", "targetXY": "1000,1000" }],
  "versions": { "mapId": "主世界地图", "currentVersion": 1, "versions": [{ "v": 1, "chapter": "第一章", "label": "初始版本", "changes": [...] }] }
}
```

## 技术栈

- 纯 HTML + CSS + JavaScript
- Canvas 2D 渲染
- File System Access API（Chrome/Edge 86+，工作目录直读直写）
- IndexedDB（持久化目录句柄）

## 向后兼容

- 旧版 `circle` 类型自动转换为 `ellipse`（`center+radius` → `cx/cy/rx/ry`）
- 旧版 `rangeMarkers` 数据通过 `migrateGeoData()` 一次性迁移到 `geoMarkers`
- 保存文件保留 `rangeMarkers: []` 空数组，旧版本加载不会报错
- 缺失的 `style`/`presetType` 字段自动补充默认值
