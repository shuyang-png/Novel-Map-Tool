# Novel-Map-Tool

纯前端小说地图编辑工具，单 HTML 文件，打开即用。JSON 文件为核心数据，HTML 为渲染器/编辑器。

## 功能列表

### 地图画布
- 2000×2000 像素画布，带网格线和坐标轴
- 鼠标滚轮缩放（0.1x ~ 2x）
- 鼠标拖拽平移
- 自适应窗口大小

### 坐标点
- 点击地图直接添加坐标点（名称 + 备注）
- 左侧面板手动输入坐标添加（X/Y 0~2000）
- 坐标点列表：查看、删除
- 点击坐标点弹出详情框，支持在线编辑备注（≤200字）

### 地理标识（V2 — 25 种预设）

地理标识用于在地图上标记各类地理特征，每种预设绑定视觉符号，不只靠颜色区分。

**25 种预设，7 个分组：**

| 分组 | 预设 |
|------|------|
| 水域 | 河流、湖泊、海洋、泉眼 |
| 植被 | 森林、沼泽、草原、沙漠、竹林 |
| 地形 | 山脉、悬崖、火山、雪峰、洞穴 |
| 建筑/人文 | 城市、城堡、村庄、庙宇、废墟 |
| 道路/边界 | 道路、城墙、行军路线、桥梁 |
| 范围/区域 | 王国、营地 |

**5 种底层形状：**
- 折线（polyline）— 线状特征
- 多边形（polygon）— 不规则区域
- 矩形（rect）— 规则区域
- 椭圆（ellipse）— 规则区域
- 标注点（pin）— 具体位置

**形状约束：**
- 线状特征（河流、山脉、道路等）只能使用折线
- 面状特征不能使用折线，但椭圆/矩形/多边形自由选择
- 标注点对所有类型开放

**Pin 快捷放置：**
- 选择标注点类型（泉眼、火山、雪峰、洞穴、村庄、庙宇、废墟）时，点击「📌 点击地图放置」按钮
- 光标变为十字，在地图上点击即可自动填入坐标
- 自动退出放置模式

**预设自动填充：**
- 选择预设类型后，描边色、线宽、粗线、内部符号自动填入
- 用户可手动覆盖任何样式参数

**视觉符号（Canvas 几何绘制，无字体依赖）：**
- 河流 → 沿线方向箭头
- 森林 → 散布小树（△ + 竖线）
- 湖泊 → 水平波浪线
- 城墙 → 实线 + T 形止标
- 营地 → 帐篷形（△ + 旗子）
- 城市 → 十字街道网格
- 等等……每种预设都有独特的内部符号

### 地图关联
- 设置当前地图与目标地图的双向关联
- 关联点在地图上以 🚪 传送门标记显示，附目标地图名称
- 自动在目标地图中创建反向关联（双写到两个 JSON 文件）
- 支持删除关联（同步删除反向并更新两个 JSON）

### 量化单位
- 自定义单位名称（如"里"）
- 自定义单位说明（如"1里=500米"）
- 显示在地图左下角

### 文件操作
- **📄 新建地图**：不刷新页面即可创建空白地图，输入名称后画布自动重置，可直接开始编辑
- **💾 保存地图**：保存当前地图（有工作目录时直写文件，无目录时降级下载）
- **📦 导出所有地图**：将所有已保存地图打包为一个 JSON
- **读取地图**：从本地 JSON 文件或工作目录加载地图数据
- **批量导入地图**：一次选择多个 JSON 文件导入
- **清空所有数据**：重置画布

### 工作目录（File System Access API）
- 选择本地文件夹作为工作目录（需 Chrome/Edge 86+）
- 保存直接写入目录文件，不再触发浏览器下载
- 目录中的 JSON 文件自动列出，点击即可加载
- 目录选择状态持久化，刷新页面后自动恢复
- 不支持的浏览器自动降级到传统下载/上传方式

### 已保存地图管理
- 左侧面板显示已保存地图列表
- 一键加载/删除已保存的地图
- 批量导入后右侧显示地图切换列表

## 使用方式

直接用浏览器打开 `小说地图工具.html`，无需安装任何依赖，无需联网。

**完全离线可用**：单个 HTML 文件，零外部依赖，不发起任何网络请求。地图数据始终在本地，适合保护创作资产。

推荐使用 **Chrome 86+** 或 **Edge 86+** 以获得工作目录直读直写功能。

## 数据格式

地图数据以 JSON 保存，核心结构：

```json
{
  "mapName": "主世界地图",
  "scale": 1,
  "mapSize": 2000,
  "unit": { "name": "里", "desc": "1里=500米" },
  "notes": [{ "id": "note_xxx", "name": "长安城", "x": 1000, "y": 800, "content": "帝都" }],
  "geoMarkers": [
    { "id": "geo_xxx", "presetType": "河流", "type": "polyline", "name": "青龙河",
      "points": [{"x":200,"y":100},{"x":500,"y":300}],
      "style": { "strokeColor": "#5dade2", "lineWidth": 3, "bold": false, "symbol": "arrows" } },
    { "id": "geo_yyy", "presetType": "城市", "type": "rect", "name": "长安城",
      "x": 800, "y": 600, "width": 200, "height": 150,
      "style": { "strokeColor": "#e74c3c", "lineWidth": 2, "fillColor": "rgba(231,76,60,0.12)", "bold": true, "symbol": "grid" } }
  ],
  "mapRelations": [{ "id": "relation_xxx", "currentMapName": "主世界", "currentXY": "200,1800", "targetMapName": "火焰秘境", "targetXY": "1000,1000" }]
}
```

## 技术栈

- 纯 HTML + CSS + JavaScript
- Canvas 2D 渲染
- FileSaver.js（CDN 引入，JSON 下载降级方案）
- File System Access API（Chrome/Edge 86+，工作目录直读直写）
- IndexedDB（持久化目录句柄）

## 向后兼容

- 旧版 `circle` 类型自动转换为 `ellipse`（`center+radius` → `cx/cy/rx/ry`）
- 旧版 `rangeMarkers` 数据通过 `migrateGeoData()` 一次性迁移到 `geoMarkers`
- 保存文件保留 `rangeMarkers: []` 空数组，旧版本加载不会报错
- 缺失的 `style`/`presetType` 字段自动补充默认值
