# Novel-Map-Tool

纯前端小说地图编辑工具，单 HTML 文件，打开即用。JSON 文件为核心数据，HTML 为渲染器/编辑器。

## 功能列表

### 地图画布
- 2000×2000 像素画布，带网格线和坐标轴
- 鼠标滚轮缩放（0.1x ~ 2x）
- 鼠标拖拽平移
- 自适应窗口大小

### 坐标点
- 点击地图直接添加坐标点（名称 + 备注）
- 左侧面板手动输入坐标添加（X/Y 0~2000）
- 坐标点列表：查看、删除
- 点击坐标点弹出详情框，支持在线编辑备注（≤200字）

### 范围标识
- 矩形区域：输入 4 个顶点坐标绘制
- 圆形区域：输入圆心 + 半径绘制
- 标识在地图上以红色实线显示，中心标注名称
- 支持编辑、删除

### 地图关联
- 设置当前地图与目标地图的双向关联
- 关联点在地图上以 🚪 传送门标记显示，附目标地图名称
- 自动在目标地图中创建反向关联（双写到两个 JSON 文件）
- 支持删除关联（同步删除反向并更新两个 JSON）

### 量化单位
- 自定义单位名称（如"里"）
- 自定义单位说明（如"1里=500米"）
- 显示在地图左下角

### 文件操作
- **💾 保存地图**：保存当前地图（有工作目录时直写文件，无目录时降级下载）
- **📦 导出所有地图**：将所有已保存地图打包为一个 JSON
- **读取地图**：从本地 JSON 文件或工作目录加载地图数据
- **批量导入地图**：一次选择多个 JSON 文件导入
- **清空所有数据**：重置画布

### 工作目录（File System Access API）
- 选择本地文件夹作为工作目录（需 Chrome/Edge 86+）
- 保存直接写入目录文件，不再触发浏览器下载
- 目录中的 JSON 文件自动列出，点击即可加载
- 目录选择状态持久化，刷新页面后自动恢复
- 不支持的浏览器自动降级到传统下载/上传方式

### 已保存地图管理
- 左侧面板显示已保存地图列表
- 一键加载/删除已保存的地图
- 批量导入后右侧显示地图切换列表

## 使用方式

直接用浏览器打开 `小说地图工具.html`，无需安装任何依赖。

推荐使用 **Chrome 86+** 或 **Edge 86+** 以获得工作目录直读直写功能。

## 数据格式

地图数据以 JSON 保存，结构如下：

```json
{
  "mapName": "主世界地图",
  "scale": 1,
  "mapSize": 2000,
  "unit": { "name": "里", "desc": "1里=500米" },
  "notes": [{ "id": "note_xxx", "name": "长安城", "x": 1000, "y": 800, "content": "帝都" }],
  "rangeMarkers": [{ "id": "marker_xxx", "name": "中原", "type": "rect", "params": { "x1": 400, "y1": 400, "x2": 800, "y2": 400, "x3": 800, "y3": 800, "x4": 400, "y4": 800 } }],
  "mapRelations": [{ "id": "relation_xxx", "currentMapName": "主世界", "currentXY": "200,1800", "targetMapName": "火焰秘境", "targetXY": "1000,1000" }]
}
```

## 技术栈

- 纯 HTML + CSS + JavaScript
- Canvas 2D 渲染
- FileSaver.js（CDN 引入，JSON 下载降级方案）
- File System Access API（Chrome/Edge 86+，工作目录直读直写）
- IndexedDB（持久化目录句柄）
